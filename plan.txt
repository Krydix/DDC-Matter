ESP32 Matter DDC Display Controller: Use ESP-IDF v5.x + esp-matter SDK on ESP32 classic.
Component 1 (DDC/Cl over I2C): Direct I2C master on GPIO21 SDA / GPIO22 SCL at 100kHz, pull-ups disabled (level shifter has them). EDID read at 12C addr 0x50(128 bytes, parse PnP ID from bytes 8-11, monitor name from descriptor blocks bytes 54-125). DDC/CI write (set VCP) at addr 0x37, packet [0x51, 0x84, 0x03, vcp_code, value_hi, value_lo, XOR checksum]. DDC/CI read (get VCP) at addr 0x37, command 0x01. Capabilities query VCP OxF3, multi-part read, parse for input values like 60(OF 11 12). No library needed, ~150 lines total. References: ddcutil src/base/ddc_packets.c (github.com/rockowitz/ddcutil), ddcci-driver-linux (gitlab.com/ddcci-
driver-linux/ddcci-driver-linux), ddcutil src/vcp/vcp_feature_codes.c. Component 2 (Monitor Database): Hardcode MCCS input labels (0x01=VGA-1, 0x02=VGA-2, 0x03=DVI-1, 0x04=DVI-2, 0xOF=DP-1, 0x10=DP-2, 0x11=HDMI-1, 0x12=HDMI-2, Ox13=HDMI-3, 0x14=HDMI-4). Runtime fetch monitor profile from ddccontrol-db (github.com/ddccontrol/ddccontrol-db) at https://raw.githubusercontent.com/ddccontrol/ddccontrol-
db/master/db/monitor/u<PNP_ID>.xml, parse caps add="..." attribute for 60(XX XX XX) input values, map to MCCS labels, cache in NVS. Fallback chain: 1) NVS user config, 2) NVS cached ddccontrol-db profile, 3) fetch from GitHub by PnP ID, 4) direct DDC capabilities query, 5) MCCS defaults. Component 3 (Matter Device): Endpoint 1 Level Control cluster 0x0008 for Brightness (VCP 0x10), Endpoint 2 Level Control cluster 0x0008 for Contrast (VCP 0x12), Endpoint 3 Mode Select cluster 0x0050 for Input switching (VCP 0x60) with 4 fixed modes "Input 1" through
"Input 4" mapped to DDC values from NVS config. Use espmatter: endpoint and esp_matter:cluster APIs. BLE + QR code commissioning (standard esp-matter). OTA via Matter OTA Requestor cluster. Reference: esp-matter examples/light/for Level Control, examples/mode_select_device/ for Mode Select. Component 4 (Web Config Ul): Post-commissioning mDNS web server using esp_http_server.h + mdns.h, advertise as http://display= switcher.local.z? Single-page HTML embedded via EMBED_FILES in Make. Shows detected monitor name + PnP ID, ddccontrol-db match status, 4 input rows (hex value + name + test button), brightness/contrast VCP overrides, refresh from database button, save to NVS button.
API endpoints: GET Japi/config, POST /api/config, POST /api/test, GET /api/detect. Component 5 (Boot Sequence): Init 12C, read EDID for PnP ID + name, load NVS user config (if exists skip auto-detect), else check NVS cached profile, else fetch ddccontrol-dbfrom GitHub, else query DDC capabilities, else MCCS defaults, then start Matter, start mDNS + web config, map Matter cluster commands to DDC writes. Project structure: main/ with app_main.cpp, ddc.c/h, edid.c/h, mccs.c/h, monitor_db.c/h, config.c/h, matter.cpp/h, webserver.c, CMakeLists.txt; frontend/ with index.html. Dependencies: ESP-IDF v5.x (github.com/espressif/esp-idf), esp-matter (github.com/espressif/esp-matter), connectedhomeip (pulled by esp-matter, github.com/project-chip/connectedhomeip), ddccontrol-db runtime fetch (github.com/ddccontrol/ddccontrol-db). No compile-time external dependencies beyond ESP-IDF + esp-matter.
